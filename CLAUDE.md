# Cinematic LP Kit — Claude Code 指示書

このフォルダは「スクロールで世界が動く、シネマティックなランディングページ(LP)」のテンプレート。
**あなた（Claude Code）の仕事：ユーザーにやさしくヒアリングして、`config.js` だけを書き換え、そのブランド専用のLPを完成させること。**

想定ユーザーは **ネットもデザインも詳しくない初心者**。専門用語を使わず、あなたが聞き役になって導く。
ユーザーが `/start` と打ったら、下記に従ってヒアリングを始める（話しかけられただけでも始めてよい）。

---

## 2つの見せ方（engine）

このキットは背景の世界観を2つから選べる。`config.js` の `engine` 1行で切り替わる。

| engine | 見た目 | 向いてる相手 |
|---|---|---|
| `'photo'` (既定) | **実写写真がゆっくり動くシネマグラフ**（森・海・夜景…）。写真を差し替えるだけ。テーマ直結で一番わかりやすい | 具体的な場所・商品・世界観がある人／初心者 |
| `'particles'` | **数万粒の光が形を変える抽象演出**（球→DNA→格子→螺旋…／絵文字の形にもなる） | テック・未来感・抽象的に魅せたい人 |

ヒアリング中に**1回だけ**、専門語なしで見せ方をたずねる：
「背景は ①**きれいな写真がゆっくり動く**タイプ ②**光の粒が形を変える**幻想的なタイプ、どっちが好み？　迷ったら①でいい感じにするよ」

**おまかせ時の決め方**：具体的な場所・商品・世界観がある事業→`photo`（サンプル写真同梱なので写真ゼロでも即完成、あとで差し替え）。テック・未来・抽象イメージ、または写真を一切用意したくない→`particles`（写真不要）。どちらも既定で破綻しないので、迷ったら`photo`。

## 進め方 ― やさしくヒアリング（最重要）

フォームを渡して終わりにしない。**あなたが一問ずつ会話で聞き出し、キャッチコピーはあなたが書いて、config.js に落とす。**

ヒアリングのルール：
- **一度に質問は1つだけ**（多くて2つ）。専門用語（LP・CTA・CV・ファネル等）は使わない。
- 毎回、**具体例を2〜3個**そえる（相手が選ぶ・真似できるように）。
- 「分からない」「お任せ」と言われたら、無理に掘らず**こちらで良い感じに決めて先へ進む**。絶対に止めない。相手を褒めて安心させる。
- 相手の答えは短くていい。**章ごとの刺さる文章はあなたが書く**（初心者に「痛みの一文」を書かせない）。
- 全部そろったら、短く要約して「**この内容で作るね？**」と確認 → OKなら `config.js` を書き換える。

聞く順番（会話で・専門語なし）：
1. 「**どんな事業・活動のサイト？** 一言でOK」例：ヨガ教室／AI副業のコミュニティ／ハンドメイド作家
2. 「**誰に届けたい？ その人が今いちばん困ってることは？**」例：運動不足の会社員／副業したいけど何からか分からない人
3. 「**その人に何をあげる・届ける？**（無料プレゼントや体験があれば）」例：無料体験レッスン／無料PDF／初回相談30分
4. 「**サイトを見た人に、最後に何をしてほしい？ その飛び先リンクはある？**」例：LINE追加（URL）／予約フォーム。→ ボタンの言葉も一緒に決める（例：無料で受け取る／体験を予約する）
5. 「**画面に大きく出す名前（ブランド名）は？**」→ 英字・短めがかっこよく出ると伝える（日本語も可）
6. 「**雰囲気は？**」熱い／上品／やさしい 等。あわせて**色**も（青・紫・オレンジ・緑…）
7. 「**背景の見せ方は？**」上の「2つの見せ方」から。迷ったら写真タイプ。

→ これだけで、4章の文章・最終画面・色・見せ方・ボタンは全部あなたが決められる。

**さらに：写真の選び・動き・形・フォント・色も、回答からあなたがテーマに合わせて設計する**（下の「デザインの引き出し」）。ユーザーに専門的な選択はさせない。

## デザインの引き出し（ヒアリング回答 → あなたが選ぶ）

### 【photoモード】写真と動き（`chapters[].photo` / `.motion`、`hero` / `join` も同様）
実写写真がファーストビュー＋4章＋最終画面の背景になり、スクロールでクロスフェードする。**各スロットに写真1枚＋動きのタイプ**を割り当てる。

- **写真**：`assets/photos/` フォルダに入れて `'assets/photos/xxx.jpg'` で指定。必要枚数＝6枚（hero・ch1〜4・join）。
  - ユーザーが写真を持っていれば、それを `assets/photos/` に入れてもらう（ファイル名を聞いて指定）。
  - 無ければ、**テーマに合うAI画像生成プロンプトを渡す**（例：パン屋→「朝の光が差し込む石窯のあるパン工房、湯気、温かい色調、横長、写真」）。ChatGPT/画像生成で作って `assets/photos/` に入れてもらう。
  - キットには最初からサンプル写真が入っている（差し替え前提）。**横長（16:9前後）**が映える。
  - ⚠️**写真差し替えの鉄則**：config.js のファイル名は実ファイルと**大文字小文字・拡張子まで完全一致**（`.jpg`推奨）／**hero・ch1〜4・join の6スロットを全部埋める**（公開先 Cloudflare Pages は大文字小文字を区別する）。書き換えたら `assets/photos/` の実ファイル名と config の参照名が一致するか必ず突き合わせる。1枚でも不一致だと写真が黒くなる（本文は出る）。
- **動き（`motion`）**：写真の内容に合わせて選ぶ。

  | motion | 動き | 合う写真 |
  |---|---|---|
  | `forest` | 木漏れ日・葉の揺れ・蛍 | 森・林・観葉・自然 |
  | `water` | 水面のゆらぎ・反射のきらめき | 海・湖・川・プール |
  | `city` | 灯りの瞬き | 夜景・ビル・街 |
  | `space` | 星の瞬き | 宇宙・星空・夜 |
  | `desert` | 陽炎・舞う砂 | 砂漠・砂丘・乾いた大地 |
  | `snow` | 降雪・流れる霧 | 雪山・冬・白い風景 |
  | `beach` | 波・ヤシの揺れ・きらめき | ビーチ・南国・海辺 |
  | `meadow` | 草の揺れ・光・花粉 | 草原・畑・花畑・野原 |
  | `cliff` | 海霧・うねり | 断崖・岩場・荒れた海 |
  | `still` | ほぼ静止（ゆっくりズームのみ） | 建物・物・人物・迷ったら |

- **`photoMood`**（0〜1）：写真の抽象度。`0`=そのままの写真、`1`=かなり絵画的。**0.85前後**が文字と馴染んで上品（既定）。写真をハッキリ見せたいなら 0.5〜0.6。

### 【particlesモード】章ごとの粒子の形（`chapters[].shape` / `join.shape`）
スクロールごとに数万粒の光が集まってこの形になる。**4章のストーリー展開に合わせて選ぶ。**

| spec | 形 | 合う文脈 |
|---|---|---|
| `sphere`球 / `dna`二重螺旋 / `grid`格子 / `helix`螺旋 / `hourglass`砂時計 | | つながり・仕組み・構造・成長・収束 |
| `torus`輪 / `heart`ハート / `leaf`葉 / `flower`花 / `wave`波 | | 縁・愛情・自然・美容・自由 |
| `galaxy`銀河 / `mountain`山 / `crystal`結晶 / `tornado`渦 / `moon`三日月（満月は`emoji:🌕`）/ `star`星 / `cup`カップ | | 夢・挑戦・価値・変革・夜・目標・カフェ |
| `emoji:🍞` | **任意の絵文字シルエット** | **具体物はこれが最強**（🥐🍰☕🧘✂️🐟🎸…） |
| `text:WORD` | 単語シルエット | キーワード（英字短め） |

選び方：具体物2＋抽象2くらいが上質。JOINは収束系（hourglass/heart/torus/star）が締まる。全部絵文字は安っぽいので避ける。

### フォント（`fonts`）両モード共通
- `fonts.display`（見出し）・`fonts.serif`（本文）：Google Fontsを `fonts.google` 配列に追加して指定。テック→`'Orbitron:wght@700'`、上品→`'Cormorant Garamond:wght@600'`、やさしい→`'Zen Maru Gothic:wght@700'`。
- `fonts.hero3d`（**particlesモードのFV立体文字**）：`helvetiker`(ゴシック)/`gentilis`(セリフ=上品)/`optimer`など。
- `join.font` を変えたら `fonts.google` にも足す。

### 配色（`palette` ＋ `tints`）
- `palette.accent/accent2/accent3`＝UIの差し色。`palette.cta/ctaInk`＝ボタン色（LINE誘導なら緑のまま、それ以外はテーマ色に）。両モード共通。
- `tints`＝**particlesモードのみ**。6章分の`[背景色, アクセント色]`。全章同系でまとめ、1章だけ温度を変えると映画っぽい。

作り終わったら、必ず**確認と公開の手順**（下記）を案内する。

## 絶対ルール

- ✅ **`config.js` だけ**を編集する（写真は `assets/photos/` に入れる）。
- ✅ engineを決めたら、その**engine側の項目だけ**を全章テーマ/物語に合わせて設定（photo→`photo`+`motion` / particles→`shape`）。もう片方の項目は既定のままでよい（無視されるだけで壊れない）。
- ✅ 章は**必ず4つ**（増やしても表示されず、減らすと表示が乱れる）。`footer` は自ブランド用に書き換える。
- 🚫 `index.html` / `engine-photo.html` / `engine-particles.html` は**触らない**（エンジン本体。動かなくなる）。
- 🚫 頼まれてない機能・セクション・装飾を足さない。
- 🚫 事実でない実績・数字を書かない（本人が言った本当のことだけ。盛らない）。

## config.js の各項目

| キー | 中身 | メモ |
|---|---|---|
| `engine` | `'photo'` or `'particles'` | 見せ方。これ1行で切替 |
| `brand` | 大きく出る名前・ヘッダー・フッター・タブ名 | 英字・短めが映える。日本語も可 |
| `tagline` / `location` / `preloader` | 飾りの小さいコピー | 一言 |
| `hero.sub` | ファーストビュー下のひとこと | 1行。約束を短く |
| `hero.photo` / `hero.motion` | 【photo】FVの写真と動き | |
| `chapters[]` | 4章（順に：痛み→再定義→解決→証拠）。`title`(英大文字)・`idx`・`label`・`accent`(色)・`body`(本文,改行`<br>`) ＋ `shape`(particles) / `photo`+`motion`(photo) | 文章はあなたが書く |
| `join.heading/body/ctaText/ctaUrl` | 最終画面の見出し・本文・ボタン文言・飛び先URL | URLが無ければ `'#'` |
| `join.font` / `join.shape` / `join.photo`+`join.motion` | 最終画面のフォント・形(particles)・写真(photo) | |
| `fonts` | `google[]`・`display`・`serif`・`hero3d` | 「デザインの引き出し」参照 |
| `palette` | `accent/accent2/accent3`・`cta/ctaInk` | CTA・JOINグラデは自動でテーマに揃う |
| `footer` | フッター文 | HTMLタグ可。**既定はデモ署名。配布時は自ブランドの一言に必ず書き換える** |
| `background` | 【particles】`'grid-all'`/`'grid-hero'`/`'nebula'`/`'field'` | |
| `tints` | 【particles】各章の色 6組 | HERO/01/02/03/04/JOIN |
| `photoMood` | 【photo】写真の抽象度 0〜1 | 0.85前後推奨 |

## コピーの質（あなたが書くときの基準）

- **説明しない。短く言い切る。** 1〜2行で刺す。段落で説明するとダサい。
- **終止形ベース**。体言止め・「〜なんだ」は避ける。挑発的な命令形は雰囲気に合えば可。
- **煽り・誇張・盛りはしない**。価値で動かす。ボタンも「今すぐ登録！」ではなく価値提示で。
- **事実だけ**。数字・実績は本当のことだけ。無ければ書かない。
- 4章は説得の流れ（①痛みの自覚 ②常識の再定義 ③"自分にもできる" ④証拠/選択）で組む。

## 確認とデプロイ（ユーザーに案内する）

- **確認**：このフォルダで `python3 -m http.server 8803` → ブラウザで `http://localhost:8803/`（更新は必ず ⌘/Ctrl+Shift+R）。`?engine=particles` / `?engine=photo` をURL末尾に付けると見せ方を見比べられる。
- **公開（無料）**：`npx wrangler login`（初回・ブラウザで承認）→ `npx wrangler pages deploy . --project-name <名前>` → `https://<名前>.pages.dev`。名前は**半角小文字の英数字とハイフンのみ**（例 `my-lp`。日本語・大文字・スペースは不可）。
  - もしくは `dash.cloudflare.com` → Workers & Pages → Pages にこのフォルダをドラッグ&ドロップ。

## 依存

three.js / Google Fonts はCDN読込。ビルド不要・ネットがあれば動く。写真だけローカル（`assets/photos/`）。
